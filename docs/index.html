运行方式

▼

选择对应数据库的脚本，在服务器上执行以下命令

# 1. 将脚本上传到数据库服务器
# 2. 以 oracle 用户执行：

$ sqlplus / as sysdba
SQL> @dbcheck.sql          -- 通用版，自动适配 10g~23ai

# 3. 执行完毕后在当前目录生成 HTML 文件
# 4. 将 HTML 文件上传到本平台即可生成报告
# Linux: oscheck.sh / AIX: oscheck_aix.sh / Windows: oscheck_win.ps1
# 将脚本放在同一目录，以 oracle 用户执行

$ sh oscheck.sh             # Linux
$ sh oscheck_aix.sh         # AIX
$ sh oscheck.sh -o orcl -a 30
PS> .\oscheck_win.ps1      # Windows

# 生成 dbcheck_主机名_日期.tar.gz（或 .zip）
# 上传到本平台选择「季检」即可
$ mysql -u root -p < dbcheck_mysql.sql > dbcheck_mysql_result.html

# 将 HTML 文件上传到本平台，选择「MySQL」即可
# 将 dbcheck_mysql.sql 和 oscheck 脚本放在同一目录
$ sh oscheck.sh             # Linux
$ sh oscheck.sh -u root -p 'password' -P 3306
PS> .\oscheck_win.ps1 -User "root" -Password "pass"

# 上传 tar.gz/zip 到本平台，选择「MySQL」+「季检」
$ psql -U postgres -f dbcheck_pg.sql

# 输出到 /tmp/dbcheck_pg_result.html
# 上传到本平台，选择「PostgreSQL」即可
# 将 dbcheck_pg.sql 和 oscheck 脚本放在同一目录
$ sh oscheck.sh
$ sh oscheck.sh -U postgres -d mydb -p 5432
PS> .\oscheck_win.ps1 -PgUser "postgres" -PgPassword "pass"

# 上传到本平台，选择「PostgreSQL」+「季检」
> sqlcmd -S localhost -i dbcheck_mssql.sql -o dbcheck_mssql_result.html

# 上传到本平台，选择「SQL Server」即可
$ sh oscheck.sh -P 'yourpassword'
$ sh oscheck.sh -U sa -P 'pass' -S localhost,1433
PS> .\oscheck_win.ps1 -Password "yourpassword"
PS> .\oscheck_win.ps1 -TrustedConnection

# 上传到本平台，选择「SQL Server」+「季检」
🔒安全保障：所有脚本仅使用 SELECT 只读查询，全部开源在 GitHub，可自行审计每一行 SQL。

常见问题

关于 DBCheck2Word 的常见疑问

巡检脚本在生产库运行安全吗？
完全安全。所有巡检脚本仅使用 SELECT 只读查询，不执行任何写操作、不创建任何对象、不申请排他锁。脚本全部开源在 GitHub，您可以审计每一行 SQL 代码。
支持哪些数据库版本？
Oracle 10g~26ai（含 RAC/DG/CDB）、MySQL 5.7+、PostgreSQL 12+、SQL Server 2016+。巡检脚本会自动适配对应版本。
在线版和桌面版有什么区别？
在线版通过网页上传文件生成报告，适合快速体验；桌面版安装在本地电脑，支持离线运行、批量处理、自定义模板和 AI 智能分析，数据完全不出内网，适合企业正式使用。
上传的巡检文件数据安全吗？
在线版：文件上传后仅在内存中处理，报告生成完毕后立即删除原始文件。桌面版：所有数据处理完全在本地完成，不连接任何外部网络。
