运行方式

▼

选择对应数据库的脚本，在服务器上执行以下命令

# 1. 将脚本上传到数据库服务器（选择与版本对应的 SQL 文件）
# 2. 以 oracle 用户执行（以 19c 为例）：

$ sqlplus / as sysdba

SQL> @dbcheck.sql          -- 通用版，自动适配 10g~23ai

# 3. 脚本执行完毕后，会在当前目录生成 HTML 文件
#    文件名格式：dbcheck_数据库名_日期.html
# 4. 将 HTML 文件下载到本地，上传到本平台即可生成报告
# 根据操作系统选择对应的 oscheck 脚本：
#   Linux  → oscheck.sh
#   AIX    → oscheck_aix.sh
#   Windows → oscheck_win.ps1

# ── Linux / AIX ──
# 将 4 个脚本文件放在同一目录，以 oracle 用户执行
# 1. 上传脚本到数据库服务器同一目录
#    dbcheck.sql（通用版，自动适配）/ oscheck.sh（或 oscheck_aix.sh）

# 2. 执行 oscheck（自动检测版本、收集 OS + DB 信息）
$ sh oscheck.sh             # Linux
$ sh oscheck_aix.sh         # AIX

# 可选参数（Linux / AIX 通用）：
#   -o 实例名    指定巡检的实例（多个用逗号分隔，如 -o orcl,testdb）
#   -a 天数      Alert 日志分析天数（默认 7 天，如 -a 30）
$ sh oscheck.sh -o orcl -a 30

# ── Windows ──
# 以管理员身份打开 PowerShell，将 4 个脚本放在同一目录
PS> .\oscheck_win.ps1
# 可选参数：
#   -OracleSIDs "orcl,testdb"    指定巡检的实例
#   -AlertDays 30                Alert 日志分析天数
PS> .\oscheck_win.ps1 -OracleSIDs "orcl" -AlertDays 30

# 执行完毕后自动在当前目录生成压缩文件
#   Linux/AIX → dbcheck_主机名_日期.tar.gz
#   Windows   → dbcheck_主机名_日期.zip
# 将压缩文件下载到本地，上传到本平台选择「季检」即可
# 1. 将 dbcheck_mysql.sql 上传到 MySQL 服务器
# 2. 以 root 或有权限的用户执行：

$ mysql -u root -p < dbcheck_mysql.sql > dbcheck_mysql_result.html

# 3. 执行完毕后在当前目录生成 HTML 文件
# 4. 将 HTML 文件上传到本平台，选择「MySQL」数据库类型即可
# 根据操作系统选择对应的 oscheck 脚本：
#   Linux   → oscheck.sh
#   AIX     → oscheck_aix.sh
#   Windows → oscheck_win.ps1

# ── Linux / AIX ──
# 将 dbcheck_mysql.sql 和 oscheck 脚本放在同一目录

# 1. 上传脚本到 MySQL 服务器同一目录
#    dbcheck_mysql.sql / oscheck.sh（或 oscheck_aix.sh）

# 2. 执行 oscheck（默认使用 root 用户）
$ sh oscheck.sh             # Linux
$ sh oscheck_aix.sh         # AIX

# 可选参数（Linux / AIX 通用）：
#   -u 用户名    MySQL 用户名（默认 root）
#   -p 密码      MySQL 密码
#   -S socket    指定 socket 路径
#   -P 端口      指定端口（默认 3306）
$ sh oscheck.sh -u root -p 'password' -P 3306

# ── Windows ──
# 将 dbcheck_mysql.sql 和 oscheck_win.ps1 放在同一目录
# 以管理员身份打开 PowerShell
PS> .\oscheck_win.ps1

# 可选参数：
#   -User "root"              MySQL 用户名（默认 root）
#   -Password "pass"          MySQL 密码
#   -Port 3306                MySQL 端口
#   -Host "localhost"         MySQL 主机
PS> .\oscheck_win.ps1 -User "root" -Password "pass" -Port 3306

# 执行完毕后自动在当前目录生成压缩文件
#   Linux/AIX → dbcheck_主机名_日期.tar.gz
#   Windows   → dbcheck_主机名_日期.zip
# 将压缩文件上传到本平台，选择「MySQL」+「季检」即可
# 1. 将 dbcheck_pg.sql 上传到 PostgreSQL 服务器
# 2. 以 postgres 用户执行：

$ psql -U postgres -f dbcheck_pg.sql

# 3. 脚本自动输出到 /tmp/dbcheck_pg_result.html
# 4. 将 HTML 文件上传到本平台，选择「PostgreSQL」数据库类型即可
# 根据操作系统选择对应的 oscheck 脚本：
#   Linux   → oscheck.sh
#   AIX     → oscheck_aix.sh
#   Windows → oscheck_win.ps1

# ── Linux / AIX ──
# 将 dbcheck_pg.sql 和 oscheck 脚本放在同一目录

# 1. 上传脚本到 PostgreSQL 服务器同一目录
#    dbcheck_pg.sql / oscheck.sh（或 oscheck_aix.sh）

# 2. 以 postgres 用户或 root 用户执行
$ sh oscheck.sh             # Linux
$ sh oscheck_aix.sh         # AIX

# 可选参数（Linux / AIX 通用）：
#   -U 用户名    PostgreSQL 用户（默认 postgres）
#   -d 数据库    连接的数据库名（默认 postgres）
#   -p 端口      指定端口（默认 5432）
#   -H 主机      指定远程主机（默认本地 socket）
$ sh oscheck.sh -U postgres -d mydb -p 5432

# ── Windows ──
# 将 dbcheck_pg.sql 和 oscheck_win.ps1 放在同一目录
# 以管理员身份打开 PowerShell
PS> .\oscheck_win.ps1

# 可选参数：
#   -PgUser "postgres"        PostgreSQL 用户（默认 postgres）
#   -PgPassword "pass"        PostgreSQL 密码
#   -PgHost "localhost"       主机地址
#   -PgPort 5432              端口
#   -PgDatabase "postgres"    数据库名
PS> .\oscheck_win.ps1 -PgUser "postgres" -PgPassword "pass" -PgPort 5432

# 执行完毕后自动在当前目录生成压缩文件
#   Linux/AIX → dbcheck_主机名_日期.tar.gz
#   Windows   → dbcheck_主机名_日期.zip
# 将压缩文件上传到本平台，选择「PostgreSQL」+「季检」即可
# 1. 将 dbcheck_mssql.sql 上传到 SQL Server 服务器
# 2. 使用 sqlcmd 执行：

> sqlcmd -S localhost -i dbcheck_mssql.sql -o dbcheck_mssql_result.html

# 3. 执行完毕后在当前目录生成 HTML 文件
# 4. 将 HTML 文件上传到本平台，选择「SQL Server」数据库类型即可
# 根据操作系统选择对应的 oscheck 脚本：
#   Linux   → oscheck.sh
#   Windows → oscheck_win.ps1

# ── Linux ──
# 将 dbcheck_mssql.sql 和 oscheck.sh 放在同一目录

# 1. 上传 2 个脚本到 SQL Server 所在 Linux 服务器同一目录
#    dbcheck_mssql.sql / oscheck.sh

# 2. 执行 oscheck.sh（需要提供 SA 密码）
$ sh oscheck.sh -P 'yourpassword'

# 可选参数：
#   -U 用户名    SQL Server 用户（默认 SA）
#   -P 密码      SQL Server 密码（必填）
#   -S 服务器    服务器地址和端口（默认 localhost）
$ sh oscheck.sh -U sa -P 'pass' -S localhost,1433

# ── Windows ──
# 将 dbcheck_mssql.sql 和 oscheck_win.ps1 放在同一目录
# 以管理员身份打开 PowerShell
PS> .\oscheck_win.ps1 -Password "yourpassword"

# 可选参数：
#   -User "sa"                   SQL Server 用户（默认 SA）
#   -Password "pass"             SQL Server 密码
#   -Server "localhost,1433"     服务器地址和端口
#   -TrustedConnection           使用 Windows 认证（无需用户名密码）
PS> .\oscheck_win.ps1 -User "sa" -Password "pass" -Server "localhost,1433"
PS> .\oscheck_win.ps1 -TrustedConnection

# 执行完毕后自动在当前目录生成压缩文件
#   Linux   → dbcheck_主机名_日期.tar.gz
#   Windows → dbcheck_主机名_日期.zip
# 将压缩文件上传到本平台，选择「SQL Server」+「季检」即可

关于 DBCheck2Word 的常见疑问

巡检脚本在生产库运行安全吗？
完全安全。所有巡检脚本仅使用 SELECT 只读查询，不执行任何写操作、不创建任何对象、不申请排他锁。脚本全部开源在 GitHub，您可以审计每一行 SQL 代码。
支持哪些数据库版本？
Oracle 10g~26ai（含 RAC/DG/CDB）、MySQL 5.7+、PostgreSQL 12+、SQL Server 2016+。巡检脚本会自动适配对应版本。
在线版和桌面版有什么区别？
在线版通过网页上传文件生成报告，适合快速体验；桌面版安装在本地电脑，支持离线运行、批量处理、自定义模板和 AI 智能分析，数据完全不出内网，适合企业正式使用。
上传的巡检文件数据安全吗？
在线版：文件上传后仅在内存中处理，报告生成完毕后立即删除原始文件，不做任何持久化存储。桌面版：所有数据处理完全在本地完成，不连接任何外部网络。
