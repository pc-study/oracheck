运行方式

选择对应数据库的脚本，在服务器上执行以下命令

# 1. 将脚本上传到数据库服务器（选择与版本对应的 SQL 文件）
# 2. 以 oracle 用户执行（以 19c 为例）：

$ sqlplus / as sysdba

SQL> @dbcheck12c.sql

# 3. 脚本执行完毕后，会在当前目录生成 HTML 文件
#    文件名格式：dbcheck_数据库名_日期.html
# 4. 将 HTML 文件下载到本地，上传到本平台即可生成报告
# oscheck.sh 会自动调用对应版本的 SQL 脚本并打包 tar.gz
# 只需将 4 个脚本文件放在同一目录，以 oracle 用户执行即可

# 1. 上传 4 个脚本到数据库服务器同一目录
#    dbcheck10g.sql / dbcheck11g.sql / dbcheck12c.sql / oscheck.sh

# 2. 执行 oscheck.sh（自动检测版本、收集 OS + DB 信息）
$ sh oscheck.sh

# 可选参数：
#   -o 实例名    指定巡检的实例（多个用逗号分隔，如 -o orcl,testdb）
#   -a 天数      Alert 日志分析天数（默认 7 天，如 -a 30）
$ sh oscheck.sh -o orcl -a 30

# 3. 执行完毕后自动在当前目录生成 tar.gz 文件
#    文件名格式：dbcheck_主机名_日期.tar.gz
# 4. 将 tar.gz 文件下载到本地，上传到本平台选择「季检」即可
# 1. 将 dbcheck_mysql.sql 上传到 MySQL 服务器
# 2. 以 root 或有权限的用户执行：

$ mysql -u root -p < dbcheck_mysql.sql > dbcheck_mysql_result.html

# 3. 执行完毕后在当前目录生成 HTML 文件
# 4. 将 HTML 文件上传到本平台，选择「MySQL」数据库类型即可
# oscheck.sh 会自动调用 dbcheck_mysql.sql 并采集 OS 信息，打包 tar.gz
# 将 dbcheck_mysql.sql 和 oscheck.sh 放在同一目录

# 1. 上传 2 个脚本到 MySQL 服务器同一目录
#    dbcheck_mysql.sql / oscheck.sh

# 2. 执行 oscheck.sh（默认使用 root 用户）
$ sh oscheck.sh

# 可选参数：
#   -u 用户名    MySQL 用户名（默认 root）
#   -p 密码      MySQL 密码
#   -S socket    指定 socket 路径
#   -P 端口      指定端口（默认 3306）
$ sh oscheck.sh -u root -p 'password' -P 3306

# 3. 执行完毕后自动在当前目录生成 tar.gz 文件
#    文件名格式：dbcheck_主机名_日期.tar.gz
# 4. 将 tar.gz 文件上传到本平台，选择「MySQL」+「季检」即可
# 1. 将 dbcheck_pg.sql 上传到 PostgreSQL 服务器
# 2. 以 postgres 用户执行：

$ psql -U postgres -f dbcheck_pg.sql

# 3. 脚本自动输出到 /tmp/dbcheck_pg_result.html
# 4. 将 HTML 文件上传到本平台，选择「PostgreSQL」数据库类型即可
# oscheck.sh 会自动调用 dbcheck_pg.sql 并采集 OS 信息，打包 tar.gz
# 将 dbcheck_pg.sql 和 oscheck.sh 放在同一目录

# 1. 上传 2 个脚本到 PostgreSQL 服务器同一目录
#    dbcheck_pg.sql / oscheck.sh

# 2. 以 postgres 用户或 root 用户执行
$ sh oscheck.sh

# 可选参数：
#   -U 用户名    PostgreSQL 用户（默认 postgres）
#   -d 数据库    连接的数据库名（默认 postgres）
#   -p 端口      指定端口（默认 5432）
#   -H 主机      指定远程主机（默认本地 socket）
$ sh oscheck.sh -U postgres -d mydb -p 5432

# 3. 执行完毕后自动在当前目录生成 tar.gz 文件
#    文件名格式：dbcheck_主机名_日期.tar.gz
# 4. 将 tar.gz 文件上传到本平台，选择「PostgreSQL」+「季检」即可
# 1. 将 dbcheck_mssql.sql 上传到 SQL Server 服务器
# 2. 使用 sqlcmd 执行：

> sqlcmd -S localhost -i dbcheck_mssql.sql -o dbcheck_mssql_result.html

# 3. 执行完毕后在当前目录生成 HTML 文件
# 4. 将 HTML 文件上传到本平台，选择「SQL Server」数据库类型即可
# oscheck.sh 会自动调用 dbcheck_mssql.sql 并采集 OS 信息，打包 tar.gz
# 将 dbcheck_mssql.sql 和 oscheck.sh 放在同一目录（Linux 环境）

# 1. 上传 2 个脚本到 SQL Server 所在 Linux 服务器同一目录
#    dbcheck_mssql.sql / oscheck.sh

# 2. 执行 oscheck.sh（需要提供 SA 密码）
$ sh oscheck.sh -P 'yourpassword'

# 可选参数：
#   -U 用户名    SQL Server 用户（默认 SA）
#   -P 密码      SQL Server 密码（必填）
#   -S 服务器    服务器地址和端口（默认 localhost）
$ sh oscheck.sh -U sa -P 'pass' -S localhost,1433

# 3. 执行完毕后自动在当前目录生成 tar.gz 文件
#    文件名格式：dbcheck_主机名_日期.tar.gz
# 4. 将 tar.gz 文件上传到本平台，选择「SQL Server」+「季检」即可
🔒 安全保障：所有脚本仅使用 SELECT 只读查询，不执行任何写操作、不创建任何对象、不影响生产数据库性能。全部脚本已开源，源码托管在 GitHub，可自行审计每一行 SQL。
